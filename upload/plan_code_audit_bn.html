ডিটেইল গ্যাপ অ্যানালাইসিস

প্ল্যান আইটেম বর্তমান অবস্থা (কী আছে) রায় কেমন হওয়া উচিত ছিল
1) Universal MCP config
  • .mcp/mcp_config.json এ HTTP MCP আছে।
  • VSCode/Cursor/Windsurf snippet আছে।
  • কিন্তু WebSocket stock-server entry নেই।
  • STOCK_SERVER_BASE_URL, STOCK_DEFAULT_MODEL placeholder নেই।
 আংশিক ঠিক একই কনফিগে ২টা server block থাকা দরকার ছিল:
  • HTTP MCP: http://localhost:3000/api/mcp/*
  • WS/SSE stock-server: ws://localhost:9998/ বা /sse + /messages
  • Universal section-এ URL, headers, register payload example
2) Provider/model fallback & mismatch
  • src/services/providerGateway.ts এ provider fallback chain আছে এবং clear error দেয়।
  • /api/chat এ request/provider/agent binding আংশিক আছে।
  • Request-level model precedence পুরোপুরি ব্যবহার হচ্ছে না।
  • Stock server এ model নাম দেখে heuristic provider নির্বাচন হয় (mismatch risk থাকে)।
 আংশিক ঠিক strict precedence লাগতো: agent model > request model > DB default model > env default। প্রতিবার selected provider/model audit-log এ persist করা উচিত।
3) Session vs Conversation vs Messages
  • Prisma schema তে ChatSession + ChatMessage আছে।
  • আলাদা Conversation মডেল নেই।
  • UI তে session reuse আছে, কিন্তু ChatGPT-style conversation thread layer আলাদা করা হয়নি।
 পুরোপুরি হয়নি আলাদা ৩ স্তর দরকার ছিল:
  • Editor Session (WS binding identity)
  • Conversation (sidebar thread)
  • Messages (conversation-এর অন্তর্ভুক্ত)
4) Streaming UX (thinking/typing)
  • src/components/dashboard/chat-panel.tsx এ chunk-by-chunk rendering আছে।
  • assistant placeholder + live append আছে (typing effect)।
  • sending state এ “Generating...” টাইপ feedback আছে।
 ঠিক আছে একইভাবে রাখতে হবে, তবে “Thinking…” state-কে event-driven করলে আরও স্থিতিশীল হবে।
5) Editors page (observability proof)
  • src/app/editors/page.tsx আছে, live clients দেখায়।
  • /api/stock/clients, /api/stock/events integration আছে।
  • কিন্তু Create Session button, agent attach, allowed tools, last tool execution summary নেই।
 আংশিক ঠিক প্রতি client row-এ দরকার:
  • Create/Attach conversation button
  • Attached agent + allowed tool list
  • last message time + last tool execution
6) Agent tools permission UI + enforcement
  • mcpService.executeTool() এ agentToolAssignment enforcement আছে।
  • WS mcp_execute-তেও agentId pass করার সুযোগ আছে।
  • কিন্তু Agents dashboard-এ assign/unassign tools UI নেই।
 আংশিক ঠিক Agents panel-এ checkbox-based tool assignment UI এবং API endpoints লাগতো (assign/unassign/list)।
7) Memory stores: Agent vs Individual
  • AgentMemory ও IndividualMemory আলাদা আছে এবং store হয়।
  • /api/chat থেকে individual extraction trigger হয় (regex based)।
  • কিন্তু per user/editor session/conversation isolation নেই; global store টাইপ।
  • “last N memories with agentId/sessionId/conversationId” proof view নেই।
 আংশিক ঠিক Individual memory-তে userId/editorSessionId/conversationId ফিল্ড যোগ করে scoped persistence করতে হতো, সাথে proof widget দরকার ছিল।
