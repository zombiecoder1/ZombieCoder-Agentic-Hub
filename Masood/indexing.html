ZombieCoder Agent System Runtime Specification

Hard Rules (অবশ্যই মানতে হবে)

  • Zero Demo Output: ব্যর্থ হলে fake/placeholder/ডেমো রেসপন্স দেওয়া নিষিদ্ধ।
  • Zero Hidden Watermark: কোনো গোপন ব্র্যান্ডিং, auto-injected signature, “Generated by …” নিষিদ্ধ।
  • Fail-Closed: DB/Tool/Provider unavailable হলে সংশ্লিষ্ট capability বন্ধ থাকবে এবং স্পষ্ট error দিতে হবে।
  • Module-first: ডক/MD রানটাইমে ব্যবহার হবে না; ডকের কনটেন্ট থেকে কোড-মডিউল তৈরি হবে।

  • BAAI bge-small-en-v1.5: models--BAAI--bge-small-en-v1.5

Agent আসলে কী? (Module Definition)

একজন এজেন্ট কোনো স্ট্যাটিক prompt না। এজেন্ট হলো:

  • Policy Module: কী করা যাবে/যাবে না (governance + safety + audit).
  • Tool Access Module: environment-এর সাথে কথা বলার controlled ক্ষমতা (read/write/search/etc) + permission gating.
  • State Resolver: config/persona/tool-config/rules কোন উৎস থেকে আসবে তার resolution chain (DB-first, fallback-second).
  • Runtime Executor: provider middleware + streaming + retry + timeout.

Data Stores (MySQL বাধ্য + indexing আলাদা)

1) MySQL 8.x (System-of-record)

Authoritative state: agents, tools, permissions, governance rules, audit logs.

  • DB ডাউন হলে: write/admin/tool-config update বন্ধ (fail-closed).
  • DB ডাউন হলেও: retrieval চালাতে চাইলে indexing store আলাদা থাকতে হবে।

2) SQLite (FTS5) - Text Index (Local-first)

লোকাল ফাইলে full-text index। MySQL ডাউন হলেও read/search capability চালু রাখা যাবে (যদি লোকাল index available থাকে)।

3) Qdrant (Local) - Vector Index

Vector similarity search। embeddings model দিয়ে chunk/embed করে Qdrant-এ push হবে। Retrieval agent শুধু Qdrant থেকে top-k এনে কাজ করবে।

MySQL Schema (Minimum Required Tables)

নিচের schema গুলো “ডেমো” না—এগুলো runtime-contract। Implementation এ exact columns align করতে হবে।

-- Agents (persona + runtime policy pointer)
CREATE TABLE agents (
  id VARCHAR(64) PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  enabled BOOLEAN NOT NULL DEFAULT TRUE,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Agent runtime config (DB-first; fallback exists in code)
CREATE TABLE agent_configs (
  agent_id VARCHAR(64) PRIMARY KEY,
  system_prompt TEXT NULL,
  identity_fixed_response TEXT NULL,
  language_policy JSON NULL,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (agent_id) REFERENCES agents(id)
);

-- Tools registry configuration (enable/disable + limits + permissions)
CREATE TABLE tool_configs (
  tool_id VARCHAR(128) PRIMARY KEY,
  enabled BOOLEAN NOT NULL DEFAULT TRUE,
  requires_auth BOOLEAN NOT NULL DEFAULT TRUE,
  max_uses_per_minute INT NULL,
  timeout_ms INT NULL,
  resource_limits JSON NULL,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- Tool permissions (role-based)
CREATE TABLE tool_permissions (
  tool_id VARCHAR(128) NOT NULL,
  role VARCHAR(64) NOT NULL,
  allowed BOOLEAN NOT NULL DEFAULT FALSE,
  PRIMARY KEY (tool_id, role),
  FOREIGN KEY (tool_id) REFERENCES tool_configs(tool_id)
);

-- Governance rules (DB-controlled)
CREATE TABLE governance_rules (
  rule_id VARCHAR(128) PRIMARY KEY,
  enabled BOOLEAN NOT NULL DEFAULT TRUE,
  severity VARCHAR(16) NOT NULL,
  rule_json JSON NOT NULL,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- Audit logs (immutable intent; do not fake)
CREATE TABLE audit_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  ts TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  session_id VARCHAR(128) NULL,
  agent_id VARCHAR(64) NULL,
  tool_id VARCHAR(128) NULL,
  action VARCHAR(64) NOT NULL,
  success BOOLEAN NOT NULL,
  error TEXT NULL,
  meta JSON NULL
);

MySQL Schema (Authoritative: zombiehub.sql)

এই প্রজেক্টে MySQL স্কিমা হিসেবে /home/sahon/Desktop/agents/doc/zombiehub.sql কে authoritative ধরা হয়েছে। এই schema থেকে যেসব অংশ runtime-এ critical:

  • providers: provider registry (type=openai/ollama/gemini/other, api_url, api_key, rate limit)
  • models: provider-bound models (context_window, max_tokens, default flag)
  • agents: agent persona/runtime config (system_prompt, model_id, server_id, temperature/top_p, max_memory_tokens)
  • conversation_memory: per-agent + per-session message log (JSON serialized in messages)
  • memory_entries + memory_relationships: long-term memory graph per agent
  • rag_documents + rag_chunks: RAG ingestion pipeline staging (chunks + metadata + created_at)
  • api_requests: request audit (endpoint/method/status_code/response_time_ms/tokens_used/model_id)
  • settings: system configuration KV
  • servers: runtime server registry/health state
  • users: auth/role-based access

Zero-demo constraint: এই টেবিলগুলোর কোনো data “seed” কে runtime behavior হিসেবে ব্যবহার করা যাবে না। seed data থাকলেও সেটা testing/dev data হিসেবে treat হবে; runtime must validate availability.

Session & Memory Isolation (Per Agent + Per Session)

স্কিমা অনুযায়ী isolation model:

  • conversation_memory: (agent_id, session_id) scoped conversation state.
  • memory_entries: agent_id scoped long-term memory (importance/access_count/last_accessed_at)
  • Global vector search: Qdrant collection-এ chunk payload এ agent_id/session_id রাখা হবে; search-এ filter দেওয়া হবে বা global search হলে filter বাদ হবে।

Qdrant Vector Design (Production Contract)

Qdrant-এ vector storage MySQL-এর বাইরে থাকবে; MySQL শুধুমাত্র metadata/staging/log রাখবে।

Collection: rag_chunks_v1
Vector size: embedding_dimension (model-dependent)
Distance: Cosine

Point payload (minimum):
  chunk_id: rag_chunks.id
  document_id: rag_chunks.document_id
  chunk_index: rag_chunks.chunk_index
  agent_id: (nullable)  -- if chunk belongs to an agent-specific memory
  session_id: (nullable)
  source: rag_documents.source
  created_at: rag_chunks.created_at
  metadata: rag_documents.metadata
  • Write path: ingest → chunk → embed → upsert to Qdrant → store mapping in MySQL (optional: qdrant_point_id).
  • Read path: any agent → query embed → Qdrant search → top-k chunks → answer synthesis (LangChain chain allowed).
  • Fail-closed: Qdrant down হলে vector retrieval বন্ধ; agent must not fabricate context.

Providers (Multi-Provider + DB-Driven)

DB table providers + models থেকে runtime provider registry build হবে। কোডে বর্তমানে provider layer:

  • Implemented: Ollama (local), OpenAI (optional via ENABLE_OPENAI=true)
  • DB-present but not implemented: Gemini provider row আছে (provider-gemini) কিন্তু TypeScript provider class নেই
  • Required to add: HuggingFace provider (embedding + optional text generation)

HuggingFace Provider Contract:
- Embedding mode: bge-large-en-v1.5 অথবা bge-small-en-v1.5 → Qdrant indexing/search embedding source
- Generation mode (optional): HF inference endpoints (যদি enable করা হয়) — otherwise disable.

Legacy vs Target Vector Store (Important)

বর্তমানে server/vector-store-service.ts Ollama embeddings + লোকাল JSON index (data/vector-store/index.json) ব্যবহার করছে। এটা Qdrant নয়।

  • Target: এই সার্ভিসকে Qdrant backend দিয়ে replace করতে হবে (ডেমো behavior নয়; real storage)।
  • Constraint: “DB/provider down” হলে response fabricate করা যাবে না; vector retrieval simply disabled/fail হবে।

Fallback Strategy (DB Down / Provider Down)

Non-negotiable behavior

  • DB Down: write/admin/config mutation বন্ধ। Tool execution যেগুলো DB-permission require করে সেগুলোও বন্ধ।
  • Provider Down: LLM response hard-fail. message: "দুঃখিত, সার্ভারটি বন্ধ আছে"
  • No demo replies: ব্যর্থতার পর কোনো “approximate” বা fabricated output নিষিদ্ধ।

DB-First Resolution Chain (Runtime)

ResolveConfig(agentId):
  1) Try MySQL agent_configs for agentId
     - if available: use DB values (system_prompt, identity_fixed_response, policies)
  2) If MySQL unavailable OR row missing:
     - use local immutable defaults (bundled in code)
  3) If both missing:
     - fail-closed (agent cannot start)

Tool Registry: What to Patch / What to Replace

  • Replace: যেসব path fake behavior দেয় (যেমন tool_call এ sleep করে Done)—এগুলো সম্পূর্ণ remove/replace করতে হবে।
  • Patch: core registry/executor/memory যেখানে contract ঠিক আছে কিন্তু DB logging / persistence missing—এগুলো patch করে বাস্তবায়ন করতে হবে।

Error Message Contract (No Demo)

Runtime failure এ ইউজার-facing response: 

দুঃখিত, সার্ভারটি বন্ধ আছে

Internal logs এ অবশ্যই root-cause (DB connection error / provider timeout / permission denied) থাকবে।

Operational Checklist (Go/No-Go)

  • Tool_call real execution: ToolRegistry → permission check → execute → audited output.
  • No placeholder strings: "example.com", "support@", "Generated by", "TODO: return template" type content purge.
  • Audit integrity: logs must reflect real input/output; fake "input"/"outputs" strings forbidden.
  • Indexing durability: SQLite FTS + Qdrant local volumes persisted.

Operational Data Flow (End-to-End)

Client (Web/Admin) -> server/ (WebSocket or REST)
  -> Resolve Provider+Model from MySQL (providers + models)
  -> Resolve Agent config from MySQL (agents)
  -> Load conversation state by (agent_id, session_id) (conversation_memory)
  -> Optional retrieval:
       1) Embed query (HuggingFace bge-large/small OR local embed model)
       2) Qdrant search (top-k)
       3) Fetch chunk metadata/source (rag_chunks + rag_documents)
  -> LangChain orchestration (core/agent-executor.ts)
  -> Tool calls via ToolRegistry (core/tool-registry.ts) + policy checks
  -> Persist:
       - api_requests (request audit)
       - conversation_memory (session state)
       - memory_entries/relationships (agent long-term memory)

Hard failure contracts (NO demo):
  - Provider unavailable: reply 'দুঃখিত, সার্ভারটি বন্ধ আছে'
  - MySQL unavailable for required action: block that action and return an explicit error
  - Qdrant unavailable: retrieval disabled; do not synthesize or fabricate context
